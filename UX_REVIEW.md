# 사용자(담당자) 관점 비판적 분석 · UI/UX 점검

담당자가 실제로 쓰는 흐름을 기준으로 **문제점**, **개선점**, **보완점**을 정리했습니다.

---

## 1. 스타일 · 시각적 일관성

### 문제점

- **인라인 스타일 다수**: `style="margin-bottom:12px"`, `style="margin-top:0"`, `style="width:100%"` 등이 scattered → 나중에 테마/반응형 수정 시 찾기 어렵고 일관성 깨지기 쉬움.
- **고정 색상**: `.btn-add`가 `#0f172a` 고정 → 다크 모드에서 배경과 구분이 잘 안 되고, primary와도 어울리지 않음.
- **포커스 제거**: `outline: none`만 있고 `:focus-visible` 대체가 없음 → 키보드 사용자·접근성에서 포커스 위치를 알기 어려움.
- **데이터 초기화 버튼**: 작은 글씨 + 회색 + 밑줄만 있어서 일반 링크처럼 보이지만, 한 번 누르면 전체 데이터 삭제. 실수 클릭 위험 있고 시각적 위험도(위험 액션) 전달이 부족함.

### 개선점

- 인라인 스타일을 `.form-row--spaced`, `.modal-actions` 등 클래스로 묶어서 CSS에서 한 곳에 관리.
- `.btn-add`는 `var(--primary)` 또는 다크 모드 대응 변수 사용.
- `:focus-visible`에서만 `outline` 또는 `box-shadow`로 포커스 링 표시.
- 데이터 초기화는 "위험한 작업" 느낌이 나도록 스타일(예: 연한 danger 색) + 확인 대화상자 유지.

## 2. 레이아웃

### 문제점

- **모바일에서 헤더 버튼**: `@media (max-width: 640px)`에서 `.header-controls > * { width: 100% }`라 "이름 추가", "명단 수정"이 풀 width → 한 줄에 하나씩만 보이고 공간을 많이 씀. 출퇴근표 상단이 더 길어져 스크롤이 길어짐.
- **캘린더 날짜 셀**: `min-height: 98px`(모바일) ~ `122px`(데스크톱)인데, 칩이 3~4개면 글자 크기 10px로 빽빽해 보이고 "오픈/마감/근태/대타"가 한눈에 구분되기 어려울 수 있음.
- **자리 배치 좌우 그룹**: `left-group` 2열, `right-group` 3열인데 화면이 좁을 때 좌석 번호 순서(1,2,6,7 / 3,4,5,8…)가 물리 배치와 다르면 담당자가 헷갈릴 수 있음. (현재는 의도된 배치이면 안내 문구로 보완 가능.)
- **상세 그리드**: 768px 이상에서 `1.1fr 0.9fr`로 좌측이 더 넓은데, "선택한 날짜 상세" 블록이 길어지면 메모 영역이 많이 아래로 밀림. 스크롤 시 맥락 놓치기 쉬움.

### 개선점

- 모바일에서 "이름 추가" / "명단 수정"은 가급적 한 줄에 두 개 작게 배치하거나, 아이콘+텍스트로 줄여서 헤더 길이 감소.
- 캘린더 셀은 "오늘" 강조는 유지하되, 칩이 많을 때만 스크롤 또는 툴팁으로 보조 정보 제공 검토.
- 자리 배치 카드 상단 또는 tip에 "실제 좌석 배치와 동일한 위치입니다" 같은 한 줄 안내 추가.
- 상세+메모가 한 화면에 안 들어가면, "선택한 날짜 상세"에 max-height + 스크롤을 두어 메모가 항상 보이도록 하는 것 검토.

## 3. UI · 인터랙션

### 문제점

- **모달 닫기 방식 불일치**: 대타 팝업·카페 자리 모달은 배경 클릭 시 닫히는데, "명단 수정", "이름 추가" 모달은 배경 클릭으로 안 닫힘. 습관이 다른 사용자는 배경을 눌러서 닫으려다 실패할 수 있음.
- **알림만 사용**: 유효성·오류가 전부 `alert()` → 화면 중앙에만 잠깐 보이고 사라져서, 모바일에서 읽다가 놓치거나 "어디서 잘못됐는지" 필드와 연결이 안 됨.
- **자리 "사용 안 함" 표시**: `::after { content: "X" }`만 있어서 스크린 리더·툴팁 없음. "몇 번 자리가 사용 안 함인지" 처음 쓰는 사람은 숫자만 보고 이해할 수 있음.
- **로테이션 시작일**: 한 번 설정해 두면 자주 안 바꾸는데, 출퇴근표 카드 상단에서 시선을 꽤 잡음. "이번 주 로테이션" 요약이 더 눈에 띄면 담당자 확인이 빠를 수 있음.
- **탭 기본값**: 저장된 탭이 없을 때 "출퇴근표"가 먼저 나오는데, 담당자가 매일 "자리 배치"만 쓴다면 매번 탭 전환이 반복됨. (저장 후엔 괜찮음.)

### 개선점

- 모든 모달 공통: 배경 클릭 시 닫기 + Esc 키로 닫기(선택) 적용해 동작 통일.
- 필드 검증 오류는 가능하면 해당 입력 옆 또는 모달 내 메시지로 표시하고, `alert`는 "네트워크 오류" 등 예외적 상황에만 사용.
- "사용 안 함" 자리에 `title="사용 안 함 (자리 N번)"` 또는 `aria-label` 추가.
- 로테이션 시작일은 접이형(예: "로테이션 설정 펼치기") 또는 하단으로 내려서, 요약 타일(오픈/마감/근태/대타)이 더 먼저 보이게 배치 검토.

## 4. UX · 업무 흐름

### 문제점

- **"이름 추가"와 출퇴근표 인원 불일치**: 출퇴근표 탭에서 "이름 추가"하면 명단(roster)에만 들어가고, 오픈/마감 선택 목록(CAFE_SCHEDULE_MEMBERS)에는 안 들어감. 담당자가 "추가했는데 출퇴근표에서 선택이 안 된다"고 느낄 수 있음.
- **"모바일 우선" 배지**: 의미가 "모바일 사용을 권장한다"인지 "모바일에서만 지원한다"인지 불명확. 사용자에게 실질적 행동 안내가 없음.
- **자리 섞기 후 피드백**: 버튼 한 번에 배치가 되지만 "섞기 완료" 토스트나 짧은 문구가 없어서, 네트워크/렌더 지연 시 "눌렀는데 반영됐나?" 하는 순간이 생길 수 있음.
- **메모 저장 시점**: 메모는 blur/변경 시 자동 저장인데, "저장됐다"는 시각적 피드백이 없음. (다른 날짜로 갔다 오면 유지되므로 저장은 되고 있음.)
- **데이터 저장 안내**: "출퇴근표 데이터는 현재 기기 브라우저에 저장됩니다"가 메모 카드 하단에만 있음. 첫 사용자는 앱 하단이나 설정 영역에서 한 번만이라도 "데이터는 이 기기에만 저장됩니다"를 보는 게 좋음.

### 개선점

- "이름 추가" 시: (1) 명단만 갱신하고, "출퇴근표의 오픈/마감 목록에는 관리자가 별도로 반영해야 합니다" 같은 툴팁/안내를 넣거나, (2) 가능하면 CAFE_SCHEDULE_MEMBERS와 동기화하는 옵션을 코드에서 검토.
- "모바일 우선" → "모바일·태블릿 권장" 등으로 문구 정리하거나, 별도 도움말로 빼기.
- 자리 섞기 성공 시 1~2초 사라지는 작은 토스트("자리 배치 완료") 표시.
- 메모 자동 저장 시 "저장됨" 같은 짧은 인디케이터(옵션).
- 데이터 저장 방식 안내를 hero 카드 또는 앱 하단에 한 줄로 노출.

## 5. 접근성 · 호환성

### 문제점

- **viewport**: `user-scalable=no`로 줌 막혀 있음. 시력이 약한 사용자는 확대가 필요할 수 있어 접근성 가이드라인과 충돌할 수 있음.
- **포커스**: 위와 같이 outline 제거만 있고 대체가 없음.
- **색 대비**: 일부 `--text-sub`, 작은 배지에서 대비가 4.5:1 미만일 수 있음(특히 연한 배경 위 회색).
- **버튼/컨트롤 라벨**: 대부분 괜찮으나, "자리 섞기", "초기화" 같은 건 툴팁이나 aria-label로 "출석 인원 기준 랜덤 배치", "배치된 이름만 지우기" 등 보조 설명이 있으면 좋음.

### 개선점

- `user-scalable=no` 제거 또는 `maximum-scale=5` 등으로 완전 고정 완화.
- `:focus-visible` 스타일 추가.
- 주요 텍스트/버튼은 대비 비율 점검 후 `--text-sub` 등 조정.
- 중요 버튼에 `title` 또는 `aria-label`로 동작 요약 추가.

## 6. 요약 · 우선 적용 권장


| 우선순위 | 항목                       | 기대 효과             |
| ---- | ------------------------ | ----------------- |
| 높음   | 포커스 표시(`:focus-visible`) | 키보드·접근성 개선        |
| 높음   | 모든 모달 배경 클릭 시 닫기         | 동작 일관성, 예상 가능한 UX |
| 높음   | 데이터 초기화 버튼 스타일·위치        | 실수 클릭 방지          |
| 중간   | "사용 안 함" 자리 툴팁/aria      | 의미 명확화            |
| 중간   | .btn-add 다크 모드 대응        | 시각적 일관성           |
| 중간   | 인라인 스타일 → 클래스            | 유지보수·일관성          |
| 낮음   | "모바일 우선" 문구 정리           | 혼동 감소             |
| 낮음   | 자리 섞기 완료 피드백             | 체감 반응성            |


이 문서는 실제 수정 시 우선순위와 이유를 담당자·개발자가 공유하기 위한 기준으로 사용할 수 있습니다.
