# 유지보수·점검 가이드 (담당자용)

담당자가 주기적으로 확인하면 좋은 항목과, 수정 시 참고할 위치를 정리했습니다.

---

## 1. 사용자(담당자) 관점 점검 체크리스트

### 1.1 출퇴근표 탭


| 항목            | 확인 방법                                   | 비고                                     |
| ------------- | --------------------------------------- | -------------------------------------- |
| 로테이션 시작 월요일   | 날짜 선택 시 오픈/마감이 2주 단위로 바뀌는지              | `ROTATION_GROUP_A`, `ROTATION_GROUP_B` |
| 캘린더 월 이동      | 이전/다음 달 버튼으로 월이 바뀌는지                    |                                        |
| 날짜 클릭 시 상세    | 선택한 날짜의 오픈조/마감조·근태·대타가 맞게 표시되는지         |                                        |
| 대타 날짜 클릭 시 팝업 | 대타가 있는 날짜 클릭 시 "대타 현황" 모달이 뜨는지, 내용이 맞는지 |                                        |
| 주말            | 토·일은 근무일이 아니므로 오픈/마감 비움 처리되는지           |                                        |
| 근태 추가/삭제      | 지각·결근·병가 등 추가 후 캘린더·상세에 반영되는지           |                                        |
| 대타 추가/삭제      | 대상자·대타자·기간·역할 입력 후 기간 내 날짜에 반영되는지       |                                        |
| 메모 저장         | 선택한 날짜 메모 입력 후 다른 날짜 갔다 와도 유지되는지        |                                        |


### 1.2 자리 배치 탭


| 항목        | 확인 방법                                            | 비고  |
| --------- | ------------------------------------------------ | --- |
| 출석 체크     | 명단에서 체크 해제 시 불참으로 카운트·표시되는지                      |     |
| 자리 섞기     | 출석 인원만 숫자 자리에 배치되는지, 카페팀 자리 2명은 제외되는지            |     |
| 카페팀 자리    | 카페팀 자리 클릭 시 인원 선택 가능, 배치 후 "자리 섞기" 시 해당 인원 제외되는지 |     |
| 사용 안 함 자리 | 숫자 자리 터치 시 잠금/해제, 자리 섞기 시 해당 자리 제외되는지            |     |
| 인원 초과 시   | 출석 인원이 사용 가능 숫자 자리보다 많을 때 안내 메시지가 나오는지           |     |
| 초기화       | 초기화 후 모든 숫자 자리 이름이 비워지는지                         |     |
| 카톡 공유     | 공유 시 캡처 영역이 이미지로 복사·전달되는지                        |     |


### 1.3 공통·설정


| 항목          | 확인 방법                                      | 비고                                            |
| ----------- | ------------------------------------------ | --------------------------------------------- |
| 테마 (라이트/다크) | 토글 시 전체 색상이 바뀌고, 새로고침 후에도 유지되는지            |                                               |
| 탭 전환        | 출퇴근표 ↔ 자리 배치 전환 후에도 선택한 탭이 기억되는지           |                                               |
| 명단 수정       | "명단 수정"에서 이름 추가/삭제/순서 변경 후 저장 시 반영되는지      |                                               |
| 이름 추가 (모달)  | 출퇴근표 탭 "이름 추가"로 이름 넣었을 때 명단·출석에 반영되는지      | 카페팀 출퇴근표 이름 목록은 `CAFE_SCHEDULE_MEMBERS` 별도 관리 |
| 데이터 초기화     | "앱이 작동하지 않나요? 데이터 초기화" 후 확인 시 모든 데이터 리셋되는지 |                                               |


### 1.4 예외·경계

- **빈 명단**: 명단이 비었을 때 자리 섞기·출퇴근표에서 오류 없이 안내만 나오는지
- **날짜 경계**: 월말·연말 넘어갈 때 캘린더·로테이션이 맞는지
- **같은 사람 대타 중복**: 같은 날 같은 역할에 대타가 여러 건 있어도 표시·삭제가 정상인지

## 2. 보안·데이터

- **localStorage**: 개발자 도구 → Application → Local Storage에서 키 목록 확인. 외부 전송 없음.
- **XSS 대비**: 이름·메모 등 사용자/저장 데이터는 화면 출력 시 `escapeHtml()`로 이스케이프 후 사용됨.
- **외부 스크립트**: html2canvas, Pretendard 폰트만 CDN 사용. 필요 시 `index.html` 상단 링크 확인.

---

## 3. 수정 시 참고 (코드 위치)

- **인원/로테이션 상수**: `index.html` 내 `DEFAULT_ROSTER`, `CAFE_SCHEDULE_MEMBERS`, `ROTATION_GROUP_A`, `ROTATION_GROUP_B`
- **좌석 ID 목록**: `SEAT_IDS`, 및 "오늘의 자리 배치" 카드 안 `id="s1"` ~ `id="s13"`, `id="cafeSeat1"`, `id="cafeSeat2"`
- **저장 키**: `Storage.keyRoster`, `Storage.keySeats`, `Storage.keySchedule` 등 (스크립트 내 `Storage` 객체)
- **대타 팝업**: `#substitutePopupModal`, `#substitutePopupTitle`, `#substitutePopupBody`, 캘린더 클릭 시 `getEffectiveSubstitutesForDate` 호출
- **자리 섞기**: `mixSeats()` — `attendees` = 출석 중 카페팀 자리 제외 인원, `availableSeats` = 사용 가능 숫자 자리

---

## 4. 프로젝트 관리

- **버전 관리**: Git 사용 시 `upload.bat` 또는 `git add` → `commit` → `push` 로 정리
- **백업**: 중요한 변경 전 브랜치 또는 복사본 권장
- **배포**: 정적 파일만 있으므로 웹 서버에 `index.html` 등 올리면 됨 (localStorage는 기기별로 유지)

이 문서는 점검 시 업데이트해 두면 다음 담당자에게 도움이 됩니다.
